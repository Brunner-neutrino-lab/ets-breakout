# Fabrication and Assembly Guide

This guide describes the process for obtaining fabricated PCBs, procuring components, and assembling the ETS 96-Channel IV-Pulse Multiplexer system.

---

## PCB Overview

The system consists of two distinct PCB designs:

| Board | KiCad Project | Qty Required |
|---|---|---|
| IV Pulse Mux board | `iv-pulse-mux.kicad_pro` | 4 |
| Motherboard | `iv-pulse-motherboard.kicad_pro` | 1 |

---

## Generating Fabrication Files

### Method 1: KiCad Job Set (Recommended)

A pre-configured KiCad job set (`fab-files.kicad_jobset`) automates generation of all required outputs:

1. Open `iv-pulse-mux.kicad_pro` in KiCad.
2. In PCB Editor (Pcbnew), go to **File → Fabrication Outputs → Run DRC** and confirm zero errors.
3. Go to **File → Fabrication Outputs → Job Set...** and open `fab-files.kicad_jobset`.
4. Click **Generate All** to produce:
   - Gerber files → `pcbfab/gerber/`
   - Drill files → `pcbfab/drill/`
   - BOM → `assembly/iv-pulse-mux-bom.csv`
   - Pick-and-place positions → `assembly/iv-pulse-mux-positions.csv`

Repeat for the motherboard project.

> **Note:** The `gerb*/` and related output directories are listed in `.gitignore` and will not be committed. Always regenerate from source before board submission.

### Method 2: Manual Gerber Export

If the job set is unavailable:

1. In PCB Editor, go to **File → Plot**.
2. Select **Gerber** format.
3. Enable all copper layers, silkscreen, solder mask, courtyard, and edge cut.
4. Set output folder to `pcbfab/gerber/`.
5. Click **Plot**, then **Generate Drill Files** and save to `pcbfab/drill/`.

### Fabrication Toolkit Options

The `fabrication-toolkit-options.json` file configures the KiCad Fabrication Toolkit plugin:

| Option | Value |
|---|---|
| All active layers exported | Yes |
| Auto-translate (origin correction) | Yes |
| Auto-fill zones before export | Yes |
| Exclude DNP components | Yes |

## PCB Specifications

These are the minimum specifications to request from the board fabricator. Verify against the actual KiCad DRC settings before ordering.

| Parameter | Value |
|---|---|
| Board material | FR4 (or better for low-leakage applications) |
| Copper weight (outer layers) | 1 oz (35 µm) |
| Minimum trace width | 0.2 mm |
| Minimum clearance | 0.15 mm (guard class) |
| Minimum via drill | 0.3 mm |
| Via diameter | 0.6 mm |
| Surface finish | ENIG (Electroless Nickel Immersion Gold) recommended for fine-pitch connectors |
| Solder mask | Both sides, green (or per lab preference) |
| Silkscreen | Both sides |
| Board thickness | 1.6 mm (standard) |

> **Note on substrate:** For measurements below ~1 nA, consider using a low-absorption substrate (e.g., Rogers 4003C or PTFE-based material) to minimize surface leakage current between the bias and sense conductors.

## Assembly Notes

### General

1. **Inspect PCBs** upon receipt: check for correct dimensions, no shorts on power pads, and confirm pad and via quality under magnification.
2. **Solder paste stencil** recommended for fine-pitch SMD components (PhotoMOS, MIC5891 if in SOIC package).
3. **Reflow soldering** for SMD components; hand soldering for through-hole relays and connectors.

### Relay Orientation

- Reed relays are polarized only in that the coil has a preferred orientation for the quiescent state. Ensure relay orientation matches the footprint silkscreen.
- The high-profile DIP housing requires sufficient pad-to-pad clearance. Verify component fits within courtyard.

### MCX Connectors

- Horizontal MCX connectors (`MCX_Molex_73415-1060_Horizontal_1.6mm-PCB`) must be aligned with the PCB edge. The 1.6 mm PCB offset is critical for panel-mount applications.
- Install MCX connectors before the board-to-board connector to allow access.

### Samtec QSE/QSH Connectors

- The QSE (socket on motherboard) and QSH (header on mux boards) connectors are precision alignment parts. Do not force mating; ensure boards are properly aligned before mating.
- The SAI alignment pin variants (`_SAI`, `_SAI-L`, `_SAI-M`) provide a keying pin to prevent mis-insertion.

### Mounting and Stacking

1. Install QSH connectors on all four mux boards.
2. Install QSE sockets on the motherboard.
3. Carefully mate each mux board perpendicular/parallel (per mechanical design) to the motherboard.
4. Secure with standoffs through mounting holes if provided.

## Inspection and Test After Assembly

### Visual Inspection

- Inspect all solder joints under a magnifier or microscope.
- Check for solder bridges on fine-pitch components (MIC5891, PhotoMOS).
- Verify relay coil flyback diodes are correctly oriented.

### Electrical Tests Before Powering Up

1. **Continuity test:** Verify 5 V and GND on the QSE connector pads are not shorted.
2. **Isolation test:** Verify no continuity between BIAS rail and SENSE rail.
3. **Resistance check on relays:** Coil resistance should match the relay datasheet (typically 500–1000 Ω for small reed relays).

### Firmware Bring-Up

1. Flash the firmware (see [Firmware Overview](../firmware/firmware-overview.md)).
2. Connect via serial terminal at 9600 baud.
3. Send `d` — should return four lines of `0` (all states zero after power-up).
4. Send `a 1` — the relay for channel 1 should close (audible click if in a quiet environment).
5. Send `d` — Board 0 state should show `1` (hex), Boards 1–3 should show `0`.
6. Send `z` — all relays should open.
7. Cycle through several channels with `a <n>` and verify relay actuation.

## Storage and Handling

- Store assembled boards in anti-static bags.
- Handle QSE/QSH connectors carefully; the alignment pins are fragile.
- Reed relays are sensitive to strong magnetic fields — store away from permanent magnets.
- If boards are to be installed in a vacuum system, allow adequate outgassing time in a low-pressure environment before sealing.
